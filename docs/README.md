# 📁 문서 관리 가이드

이 디렉토리는 프로젝트의 개발 관련 문서들을 체계적으로 관리하기 위한 공간입니다.

## 📋 디렉토리 구조

```
docs/
├── README.md                         # 이 파일 (문서 관리 가이드)
├── requirements/                     # 기능 개발 요구서
│   ├── template.md                  # 요구서 작성 템플릿
│   ├── YYYY-MM-DD_기능명.md         # 개별 요구서 (날짜_기능명 형식)
│   └── completed/                   # 완료된 요구서 아카이브
├── development/                     # 개발 과정 문서
│   ├── implementation_log.md        # 구현 과정 기록
│   └── testing_results.md           # 테스트 결과 기록
├── architecture/                    # 시스템 아키텍처 문서
│   ├── database_design.md           # 데이터베이스 설계
│   ├── api_specification.md         # API 명세
│   └── system_overview.md           # 시스템 전체 구조
└── PPT 발표 자료/                    # 🆕 프로젝트 소개 PPT 가이드
    ├── PPT_개발과정_소개_구성안.md   # PPT 슬라이드 구성안 (23개 슬라이드)
    ├── PPT_이미지_및_자료_목록.md    # 필요한 이미지/차트 목록
    └── PPT_한눈에_보기.md            # 빠른 참조용 요약
```

## 🎯 사용 방법

### 1. 새로운 기능 개발 요청
1. `requirements/template.md` 파일을 복사
2. `requirements/YYYY-MM-DD_기능명.md` 형식으로 파일명 작성
3. 요구사항을 상세히 작성
4. AI에게 검토 요청

### 2. 개발 진행 중
- 구현 과정은 `development/implementation_log.md`에 기록
- 테스트 결과는 `development/testing_results.md`에 기록

### 3. 완료 후
- 완료된 요구서는 `requirements/completed/`로 이동
- 시스템 변경사항이 있으면 `architecture/` 문서 업데이트

## 📝 다음 단계

첫 번째 기능 개발 요구서를 작성하려면:
1. `docs/requirements/template.md` 파일 확인
2. 새로운 요구서 파일 생성
3. 요구사항 작성 후 AI에게 검토 요청

## 🎤 프로젝트 발표 자료

프로젝트를 외부에 소개하거나 발표해야 할 때 참고할 수 있는 PPT 가이드가 준비되어 있습니다.

### 📊 PPT 발표 자료 가이드

1. **[PPT_개발과정_소개_구성안.md](./PPT_개발과정_소개_구성안.md)**
   - 23개 슬라이드의 상세한 구성안
   - 각 슬라이드별 내용, 시각 요소, 발표 메모 포함
   - 서론부터 향후 계획까지 완전한 스토리라인
   - 발표 시간: 약 20분

2. **[PPT_이미지_및_자료_목록.md](./PPT_이미지_및_자료_목록.md)**
   - 필요한 스크린샷 체크리스트 (19개)
   - 다이어그램 템플릿 및 제작 가이드 (6개)
   - 차트/그래프 데이터 예시 (4개)
   - 추천 도구 및 무료 리소스 사이트

3. **[PPT_한눈에_보기.md](./PPT_한눈에_보기.md)**
   - 빠른 참조용 요약 (30초 버전)
   - 암기해야 할 핵심 숫자
   - 예상 질문 & 답변 (8개)
   - 발표 당일 체크리스트

### 🎯 사용 시나리오

- **학교 관계자 대상**: 교육적 성과와 사용 편의성 강조
- **투자자 대상**: 기술적 우수성과 확장 가능성 강조
- **교육 컨퍼런스**: 도전 과제 해결 과정과 교육적 효과 강조
- **개발자 커뮤니티**: 기술 스택과 아키텍처 중심 발표
