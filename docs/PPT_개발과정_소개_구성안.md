#### 슬라이드 5: 기술 스택 개요
**제목**: 어떤 기술로 만들었을까?

**내용**:
```
┌─────────────────────────────────────┐
│       프론트엔드 (사용자 화면)        │
│    React + TypeScript + Tailwind    │
└─────────────────────────────────────┘
              ↕ API 통신
┌─────────────────────────────────────┐
│         백엔드 (서버/API)            │
│       Node.js + Express.js          │
└─────────────────────────────────────┘
              ↕
┌─────────────────────────────────────┐
│    외부 서비스                       │
│  Supabase | Gemini AI | Vercel      │
└─────────────────────────────────────┘
```

**왜 이 기술들을 선택했는가?**
- **React**: 컴포넌트 기반으로 재사용성 높음
- **TypeScript**: 타입 안정성으로 버그 사전 방지
- **Node.js**: JavaScript로 프론트/백엔드 통합 개발
- **Supabase**: 빠른 데이터베이스 구축 + 실시간 기능
- **Gemini AI**: 경제적이고 강력한 AI 분석

**발표 메모**:
"빠른 개발과 안정성, 그리고 확장 가능성을 고려해 최신 웹 기술을 선택했습니다."

#### 슬라이드 7: 백엔드 - 서버 및 API
**제목**: 백엔드: Node.js + Express

**내용**:
**⚙️ 서버 구조**
- **Node.js + Express**: JavaScript 기반 서버
- **RESTful API**: 명확한 엔드포인트 설계

**📡 주요 API**
```
/api/auth        ← 교사 로그인/회원가입
/api/rooms       ← 활동방 생성 및 관리
/api/responses   ← 학생 응답 제출 및 조회
/api/analyze     ← AI 분석 요청
```

**🔐 보안 기능**
- JWT 기반 인증
- Row Level Security (데이터 접근 제어)
- 입력 검증 및 XSS 방지

**시각 요소**:
- API 흐름도 다이어그램
- 인증 과정 시각화

**발표 메모**:
"안정적인 서버 아키텍처로 데이터를 안전하게 처리하고, 빠른 응답 속도를 보장합니다."

#### 슬라이드 8: 데이터베이스 설계
**제목**: 데이터베이스: Supabase (PostgreSQL)

**내용**:
**🗄️ 주요 테이블**
```
Teachers (교사)
  └── Activity_Rooms (활동방)
       └── Student_Responses (학생 응답)
            ├── AI_Analysis (AI 분석)
            └── Teacher_Evaluations (교사 평가)
```

**💎 핵심 기능**
- **실시간 구독**: 학생이 제출하면 교사 화면에 즉시 표시
- **JSONB 저장**: 유연한 응답 데이터 구조
- **자동 백업**: 데이터 손실 방지
- **RLS (Row Level Security)**: 교사는 본인 데이터만 접근

**📊 통계**
- 7개 핵심 테이블
- 평균 응답 시간 < 100ms

**시각 요소**:
- 데이터베이스 ERD (개체-관계 다이어그램)
- 실시간 구독 흐름도

**발표 메모**:
"Supabase 덕분에 복잡한 서버 인프라 없이도 실시간 기능을 구현할 수 있었습니다."

#### 슬라이드 9: AI 통합 - Google Gemini
**제목**: AI 분석: Google Gemini API

**내용**:
**🤖 AI의 역할**
1. 학생 응답 자동 분석
2. 사고 깊이 평가 (구체성, 논리성, 창의성)
3. 맞춤형 피드백 생성
4. 성장 추적 지원

**💡 분석 기준**
- ✅ 내용의 적절성 (각 단계 목적에 부합하는가?)

- ✅ 깊이 (표면적이지 않고 깊이 있는 사고인가?)
- ✅ 창의성 (독창적이고 다양한 관점인가?)

**📈 성과**
- 평균 신뢰도: 85%
- 분석 시간: 2-3초
- 비용: 응답당 약 $0.001 (경제적!)

**시각 요소**:
- AI 분석 결과 예시 화면
- 평가 기준 차트

**발표 메모**:
"AI가 교사의 피드백 부담을 줄이고, 학생들에게 즉각적인 학습 지원을 제공합니다."

#### 슬라이드 10: 기능 1 - 활동방 생성 및 코드 시스템

**🎯 어떻게 작동하나요?**
1. **교사**: 활동방 생성 → 자동으로 6자리 고유 코드 생성
2. **학생**: 코드 입력만으로 즉시 참여 (로그인 불필요!)

#### 슬라이드 13: 기능 4 - AI 자동 분석 및 피드백

**내용**:
**🤖 AI가 분석하는 것**

**1. 종합 평가 점수** (0-100점)
- 각 단계별 품질 평가
- 전체적인 사고 깊이 측정

**2. 세부 분석**
```
✅ 강점

- "창의적인 질문을 제시함"

💡 개선점
- "더 다양한 관점에서 생각해보세요"
- "근거를 더 명확히 제시하면 좋겠어요"
```

**3. 맞춤형 피드백**
- 학생 수준 고려
- 사고루틴 유형별 특화 분석
- 성장 가능성 제시

**📊 실제 효과**
- 교사 피드백 시간 60% 절감
- 학생 즉각 피드백으로 학습 동기 향상

**시각 요소**:
- AI 분석 결과 화면 예시
- Before/After (AI 없을 때 vs 있을 때)

**발표 메모**:
"AI가 교사를 대체하는 것이 아니라, 교사가 더 깊이 있는 피드백에 집중할 수 있도록 돕습니다."

#### 슬라이드 14: 기능 5 - 학생 포트폴리오
**제목**: 핵심 기능 ⑤ 개인별 학습 포트폴리오

**내용**:
**📚 포트폴리오 기능**
1. **모든 활동 기록**
   - 참여한 활동방 목록
   - 제출한 응답 모음
   - 날짜별 정리

2. **성장 추적**
   - AI 평가 점수 변화 그래프
   - 사고루틴별 참여 횟수
   - 강점 영역 파악

3. **반성적 학습**
   - 이전 응답 다시 보기
   - 성장 과정 확인
   - 메타인지 개발

**💡 학생 참여 유형**
- **오프라인 참여**: 활동은 오프라인, 제출은 온라인
- **온라인 참여**: 전체 활동을 온라인으로 수행
